# Serial Monitor v1.1.2

출시일: 2026-07-15

## 로그 저장과 파일 관리

- 앱 시작과 프로필 불러오기 시 로그 저장을 항상 OFF로 초기화해 의도하지
  않은 파일 기록을 방지합니다.
- `LOG ON`을 누를 때마다 새 로그 파일을 만들며 기존 파일에 이어 쓰지
  않습니다.
- 로그 저장이 꺼진 상태에서 정확한 파일명을 지정할 수 있습니다. 잘못된
  Windows 파일명과 기존 파일 덮어쓰기는 명시적으로 거부합니다.
- 자동 파일명 충돌, 날짜 변경, 크기 회전을 안전하게 처리하고 현재 파일
  경로와 기록 상태를 UI와 진단 정보에 표시합니다.
- 이벤트 감지 결과는 bounded UI 버퍼에서 관리하고 별도의 이벤트 로그
  파일은 만들지 않도록 구조를 단순화했습니다.

## 터미널과 표시 설정

- 타임스탬프 표시 형식을 날짜/시간 및 밀리초 포함 여부에 따라 네 가지로
  선택할 수 있습니다.
- 표시 형식을 바꾸면 bounded 로그 화면을 새 형식으로 다시 구성하며 파일
  로그의 원본 데이터 흐름과 분리합니다.
- xterm 화면에서 Enter 키로 자동 스크롤을 전환할 수 있고 키 반복 입력으로
  상태가 연속 변경되지 않도록 했습니다.
- 로그 저장 상태와 파일명 입력 가능 여부를 더 명확하게 표시합니다.

## 규칙 처리와 MOCK 안정성

- 이벤트, 강조, 보기 필터가 동일한 방향 및 모드 판정 정책을 사용하도록
  규칙 매칭을 통합했습니다.
- Terminal 규칙과 HEX 규칙을 분리하고 HEX 데이터는 `RawBytes`를 기준으로
  정확하게 비교합니다.
- MOCK 스트레스 데이터 분할을 별도 구성 요소로 분리해 여러 레코드가 한
  청크에 들어오거나 레코드가 청크 경계에서 나뉘는 경우를 안정적으로
  처리합니다.
- 프로필 복원 시 브리지와 로그 저장 같은 런타임 동작을 자동으로 켜지
  않도록 안전 기본값을 적용했습니다.

## Raw COM 브리지

- 물리 RX 콜백이 브리지 큐를 기다리지 않도록 bounded 비동기 전달 경로를
  정비했습니다. 용량 초과는 장치 연결이 아니라 해당 브리지 세션만
  중지시키고 원인을 진단 정보로 노출합니다.
- 장치에서 관측한 수신 간격을 가상 포트 출력에 최선 방식으로 재현하며
  지연과 초과 시간을 계측합니다.
- 브리지 트래픽 우선순위, 전역 idle guard, 단일 수동 TX 대기 슬롯을 갖는
  물리 TX 스케줄러를 추가해 바이트 스트림의 교차 전송을 방지합니다.
- 수동 TX 상태 변화를 별도 이벤트로 전달해 Waiting/Sending 동안 모든 TX
  진입점을 즉시 비활성화합니다.
- 가상 포트 TX 표시 로그를 bounded 백그라운드 파이프라인에서 처리하고
  Terminal 디코딩과 최대 256바이트 HEX 레코드를 지원합니다.
- 큐 바이트 수, 가장 오래된 항목의 대기 시간, overflow 원인, 최근 활동,
  수동 TX 중재 상태를 진단 정보에 추가했습니다.

## 검증

- Core 테스트 35개와 WinUI 테스트 95개, 총 130개를 통과했습니다.
- COM4/COM5 com0com 양방향 SHA-256 일치, 수동 TX 중재/취소, 브리지 전용
  overflow 오류 경로를 자동 검증했습니다.
- Release 빌드를 경고 0개, 오류 0개로 완료했습니다.
- `git diff --check`를 통과했습니다.

## 프리릴리스 안내

- 이번 빌드는 USB-UART 어댑터 및 드라이버별 실장비 장시간 검증이 남아
  있어 프리릴리스로 제공합니다.
- 브리지 HEX 표시 레코드 경계를 가로지르는 패턴은 매칭하지 않습니다.
  Raw 브리지 전송 자체는 표시 레코드 처리와 독립적입니다.
- 설치 파일은 코드 서명이 없어 Windows SmartScreen에서 알 수 없는
  게시자 경고가 표시될 수 있습니다.
