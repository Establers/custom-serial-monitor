# Serial Monitor v1.1.3

출시일: 2026-07-15

## Pause View 안정성

- Pause 클릭 이후 새 로그는 화면 대기열과 retained buffer에 넣지 않으며,
  클릭 전에 승인된 xterm 작업만 모두 비운 뒤 화면을 완전히 정지합니다.
- RX, 파싱, 이벤트 검출은 Pause 중에도 계속되고 파일 저장은
  `Keep saving file log` 옵션에 따라 독립적으로 동작합니다.
- Pause 구간 로그는 Resume, 필터 변경, 전체 재렌더링, 최소화/복원 이후에도
  화면에 다시 나타나지 않습니다.
- 파일 저장과 화면 표시가 같은 Pause 경계 판정을 사용해 전환 시점의
  화면/파일 불일치를 방지합니다.
- xterm drain이 30초 안에 끝나지 않으면 Pause를 무한 대기시키지 않고
  명시적인 실패 경계를 남긴 뒤 Live 상태로 복구합니다.

## 상태 표시

- Pause 정책으로 의도적으로 생략한 줄은 `PS`로 별도 집계합니다.
- `Drop UI`는 실제 UI 과부하로 유실된 로그만 집계합니다.
- Resume 시 `VIEW RESUMED - PS ...` 시스템 경계를 표시하고, 모든 SYS 로그는
  xterm에서 회색으로 표시합니다.
- 하단 상태창의 수신 단위를 bytes/lines 대신 `B`/`l`로 간결하게 표시합니다.
- PS 및 누락 카운터는 64비트 누적값으로 유지합니다.

## 검증

- Core 테스트 35개와 WinUI 테스트 110개, 총 145개를 통과했습니다.
- Pause 중 20,000개 레코드가 화면 backlog를 만들지 않으면서 파일 저장
  대상으로 모두 유지되는 정책 테스트를 통과했습니다.
- Pause 구간 레코드가 retained buffer와 전체 재렌더링 스냅샷에 포함되지
  않는 테스트를 통과했습니다.
- Release 빌드를 경고 0개, 오류 0개로 완료했습니다.
- `git diff --check`를 통과했습니다.

## 프리릴리스 안내

- 실제 USB-UART 어댑터별 장시간 Pause/Resume 및 파일 저장 검증이 남아 있어
  프리릴리스로 제공합니다.
- 설치 파일은 코드 서명이 없어 Windows SmartScreen에서 알 수 없는 게시자
  경고가 표시될 수 있습니다.
